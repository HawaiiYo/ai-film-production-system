## 2. 测试一：快速冒烟测试（5分钟）

**目的**：验证整个系统能否正常启动并产生基本输出。

### 测试步骤

#### Step 1: 验证主技能入口

在 Claude Code 中输入：

```
AI电影工坊，你好
```

**预期结果**：
- ✅ 系统以电影从业者视角回应
- ✅ 提到7个Agent或虚拟剧组概念
- ✅ 输出包含"需要我帮你策划一部什么样的电影？"类似的引导

**如果失败**：
- 检查 SKILL.md 文件是否在项目根目录
- 确认 `name: ai-filmmaking-studio` 的 name 字段

#### Step 2: 验证最简单的需求解析

输入：
```
电影策划：一个关于时间循环的5分钟科幻短片
```

**预期结果**：
- ✅ 输出 `📋 AI电影项目分析` 块
- ✅ 正确识别类型为「科幻」
- ✅ 包含情绪基调判断
- ✅ 包含复杂度评估

#### Step 3: 验证导演风格匹配

输入：
```
导演风格匹配：科幻悬疑片，主题是时间和记忆
```

**预期结果**：
- ✅ 推荐诺兰、王家卫、维伦纽瓦风格等
- ✅ 输出视觉风格指南
- ✅ 包含色调方案建议

#### 冒烟测试通过标准

| 测试项 | 通过标准 |
|--------|---------|
| 系统激活 | 以电影从业者视角回应 |
| 需求解析 | 输出结构化项目分析 |
| 导演匹配 | 输出导演组合+风格指南 |

## 3. 测试二：完整端到端流程（30分钟）

**目的**：验证从故事概念到最终AI生成指令的完整7 Agent流水线。

### 测试输入

```
AI电影工坊：我想创作一部科幻悬疑短片。

故事概念：在近未来世界，一名女科学家发现时间并非线性流动，而是层层叠加的"回响"。
她制造了一台能观测时间回响的装置，却发现自己看到的每一个"过去"都在改变"现在"。

要求：完整的制作方案，包括导演风格、剧本、角色设计、场景设计、分镜和AI生成指令。
```

### 各阶段验证点

#### Phase A: Executive Producer（流程统筹）

**验证点**：
- [ ] 输出项目分析（类型/主题/情绪基调/规模）
- [ ] 组建虚拟剧组（列出激活的Agent）
- [ ] 建立全局风格手册（色调/光影/叙事风格/技术标准）
- [ ] 判断复杂度并说明Agent激活数量

#### Phase B: Director Style（导演风格）

**验证点**：
- [ ] 推荐3-5位导演组合
- [ ] 说明匹配理由
- [ ] 输出视觉风格指南（色调/光影/构图/质感）
- [ ] 包含具体的HEX色彩代码
- [ ] 推荐协作模式

#### Phase C: Screenplay（剧本）

**验证点**：
- [ ] 输出200-300字故事大纲
- [ ] 三幕或五幕结构设计
- [ ] 场景分解（每个场景有时间/地点/功能）
- [ ] 人物列表（含人物弧光）
- [ ] 人物关系图
- [ ] 每个场景有"视觉化"描述

#### Phase D: Character Design（角色设计）

**验证点**：
- [ ] 为每个角色建立完整的角色圣经
- [ ] 面部特征包含：脸型、眼睛、鼻子、嘴、发型、特殊标记
- [ ] 服装描述包含：款式、颜色、材质、层次
- [ ] 包含7种表情描述（常态/高兴/愤怒/悲伤/惊讶/恐惧/沉思）
- [ ] 体态动作描述（站姿/走姿/习惯动作/标志性姿态）
- [ ] 输出英文 GPT-image-2 角色设计图 prompt
- [ ] 输出英文 角色固定描述片段（供复用）
- [ ] Prompt中包含画质参数（8K, hyperrealistic等）

#### Phase E: Scene & Prop（场景道具）

**验证点**：
- [ ] 为每个场景建立完整的场景圣经
- [ ] 色彩方案包含主色调/辅助色/点缀色/阴影色/高光色
- [ ] 光源详解（主光/辅光/环境光/特殊光源）
- [ ] 空间布局描述（尺寸/地面/墙面/天花板/家具）
- [ ] 材质与纹理描述
- [ ] 关键道具清单（含剧情功能说明）
- [ ] 输出英文 GPT-image-2 场景概念图 prompt
- [ ] 输出英文 场景固定描述片段（供复用）

#### Phase F: Storyboard（分镜故事板）

**验证点**：
- [ ] 将每个场景分解为逐镜分镜
- [ ] 每个镜头包含：景别/运动/构图/时长/转场
- [ ] 输出镜头总览表
- [ ] 每个镜头有详细的画面描述（中文）
- [ ] 每个镜头有英文 Seedance prompt
- [ ] 图生视频/文生视频模式选择合理
- [ ] 有情绪曲线图（ASCII艺术）
- [ ] 有拍摄顺序建议

#### Phase G: Prompt Engineer（Prompt工程）— 最终输出

**验证点**：
- [ ] 汇总所有上游Agent产出
- [ ] 输出完整性校验报告
- [ ] 优化所有 GPT-image-2 prompt
- [ ] 所有prompt末尾追加全局Style Anchor
- [ ] 执行一致性交叉检查
- [ ] 自动修正角色/场景描述不一致
- [ ] 为每类prompt添加特定Negative Prompt
- [ ] 建议技术参数（分辨率/帧率/Seed值管理）
- [ ] 输出完整 AI Generation Sheet（9个Part）
- [ ] 输出生成顺序建议

### 端到端测试通过标准

| 阶段 | 最少通过项 | 关键指标 |
|------|-----------|---------|
| Executive Producer | 4/4 | 项目分析+剧组+风格手册完整 |
| Director Style | 5/5 | 导演组合+风格指南+色调方案 |
| Screenplay | 5/5 | 结构+场景+人物+关系图 |
| Character Design | 8/10 | 角色圣经+GPT-image-2 prompt+固定片段 |
| Scene & Prop | 7/9 | 场景圣经+prompt+固定片段+道具 |
| Storyboard | 7/8 | 分镜表+Seedance prompt+情绪曲线 |
| Prompt Engineer | 8/9 | 一致性检查+优化prompt+Generation Sheet |

### 4.6 🎬 Storyboard 深度测试

#### 测试用例 6.1：镜头语言正确性

**检查分镜输出中的景别使用**：
- [ ] 场景开头使用 ELS/LS（建立空间）
- [ ] 建立镜头时长4-8秒
- [ ] 景别有变化（不连续3个以上相同景别）
- [ ] 特写/CU用于情感关键时刻

#### 测试用例 6.2：镜头运动合理性

**检查镜头运动**：
- [ ] 每个镜头不超过1种主体运动
- [ ] 运动方式与情绪匹配
- [ ] 有静态镜头穿插（静态在Seedance中最稳定）

#### 测试用例 6.3：Seedance Prompt质量

对每条视频prompt检查：
- [ ] 使用纯英文
- [ ] 包含主体/动作描述
- [ ] 包含镜头运动描述
- [ ] 包含时长（Ns, 24fps）
- [ ] 包含画质要求
- [ ] 图生视频模式标注 `Image-to-video mode`
- [ ] 包含Negative Prompt
- [ ] 动作从静态参考图自然过渡

#### 测试用例 6.4：分镜布局系统

```
输入：分镜设计：使用4格板展示关键节拍
输入：分镜设计：使用宽幅横板展示史诗场景
输入：分镜设计：需要俯视走位平面图
```

**验证点**：
- [ ] 4格板 → 2×2布局，画面大，细节多
- [ ] 宽幅横板 → 2.35:1画框比例
- [ ] 走位平面图 → 俯视视角+编号标记

### 4.7 🔧 Prompt Engineer 深度测试

- [ ] 是否检测到场景描述不一致并自动修正？
- [ ] 是否确保所有prompt包含Style Anchor？

#### 测试用例 7.2：Negative Prompt生成

检查输出中的Negative Prompt：
- [ ] 通用Negative（GPT-image-2 + Seedance各一套）
- [ ] 类型特定Negative（科幻/悬疑/古装等）
- [ ] 角色特定Negative（性别/年龄/体型防混淆）

**预期行为**：
