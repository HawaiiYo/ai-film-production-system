# 🎬 分镜连贯性规则参考

> Continuity Rules Reference — 确保镜头之间的视觉/空间/动作/情绪连贯性

---

## 一、核心连贯性原则

### 1.1 什么是分镜连贯性

分镜连贯性（Shot Continuity）是指在连续镜头之间，保证：
- **空间连贯**：角色和物体在画面中的位置关系保持一致
- **视线连贯**：角色的视线方向跨镜头一致
- **动作连贯**：角色的动作在镜头之间流畅衔接
- **光影连贯**：光源方向和性质跨镜头一致
- **时间连贯**：镜头间的时间流逝合理

违反连贯性会导致观众**出戏**——潜意识中感到"有什么不对"。

## 二、180度规则（轴线法则）

### 2.1 基本定义

两个角色（或角色与关键物体）之间有一条虚拟的**动作轴线（Action Axis / 180° Line）**。摄影机必须始终保持在轴线的**同一侧**。

```
正确的拍摄（始终在北侧）：

           北侧 (OK)
        CAM1 ●         CAM2 ●        CAM3 ●
           ╲              │              ╱
            ╲             │             ╱
             ╲            │            ╱
              ╲           │           ╱
    角色A ●══════════════════════════● 角色B
              ╱           │           ╲
             ╱            │            ╲
            ╱             │             ╲
           ╱              │              ╲
           南侧 (禁止)
```

### 2.2 越轴（Crossing the Line）

越轴时，观众会感到角色位置突然**翻转**——A原本面向右，突然面向左。

```
越轴的效果：

CAM1（北侧）：角色A在左，面朝右
CAM2（南侧·越轴）：角色A在右，面朝左 ← 观众困惑！

观众感知："A怎么突然换边了？"
```

### 2.3 安全越轴策略

如果必须越轴，使用以下方式过渡：

**策略1：中性镜头过渡**
```
镜头N（北侧）：   A面朝右 ←→ B面朝左
镜头N+1（中性）： 从一个角色的正面或背面拍摄（没有左右方向感）
镜头N+2（南侧）：  A面朝左 ←→ B面朝右（成功越轴）
```

**策略2：摄影机运动穿过轴线**
```
镜头N（北侧）→ 摄影机缓慢环移到南侧 → 镜头N+1（南侧）
（观众看到摄影机移动过程，理解空间关系变化）
```

**策略3：角色移动改变轴线**
```
镜头N：角色A起身走到B的另一侧 → 动作轴线改变
镜头N+1：在新轴线的同侧拍摄
```

## 三、视线匹配（Eyeline Match）

### 3.1 基本规则

在正反打镜头（Shot-Reverse Shot）中，两个角色的视线方向必须**相互匹配**：

```
角色A（面右看）→ → → ← ← ←（面左看）角色B

CAM1: 过肩镜头(A肩→B脸)，B面朝左 ←
CAM2: 过肩镜头(B肩→A脸)，A面朝右 →
```

### 3.2 高低差视线

```
高个子角色俯视（视线向下↘）→ → → ← ← ←（视线向上↖）矮个子角色

CAM1: 微俯拍，A面朝右下 ↘
CAM2: 微仰拍，B面朝左上 ↖
```

### 3.3 视线方向标注

在分镜表中使用以下标注：
- `LOOK→` / `LOOK←` / `LOOK↑` / `LOOK↓`：角色视线方向
- `LOOK OFF-SCREEN L/R`：看向画面外左侧/右侧
- `LOOK DIRECT-TO-CAMERA`：直视镜头（打破第四面墙）
- `EYELINE: A↔B`：角色A与B的视线匹配

## 四、角色位置追踪

### 4.1 场景空间坐标系

每个场景建立统一的坐标系统：

```
场景坐标系示例——"废弃钟楼内部"

         N(北/y+)
         ↑
         |
    ┌────┴────┬──────────┐
    │  stained │          │
    │  glass   │  巨大齿轮 │
    │  window  │  (5, 3)  │
    │  (0, 4)  │          │
    ├─────────┤          │
    │ 入口     │          │
    │ (0, 0)  │          │
    │         │  Maya    │
    │  lantern│  (3, 1)  │
    │  (1, 1) │          │
    └─────────┴──────────┘
    x+ (E/右) →

原点(0,0)：入口门中心
x轴：平行于入口墙，向右为正
y轴：垂直于入口墙，向内/北为正
z轴：垂直向上（高度），地面=0
```

### 4.2 角色位置追踪表

在分镜表中为每个镜头记录角色位置：

| 镜号 | 角色 | 位置(x,y) | 朝向 | 姿势 | 备注 |
|------|------|----------|------|------|------|
| 001 | Maya | (3, 1) | 面北 | 站立，手持手电 | 看向齿轮 |
| 002 | Maya | (3, 1.5) | 面北 | 向前迈出一步 | 动作衔接001 |
| 003 | Maya | (3.5, 2) | 面东北 | 走近齿轮 | 连续移动 |
| 004 | Maya | (3.5, 2) | 面北→面西 | 转身 | 听到声音 |

### 4.3 位置一致性检查

**规则**：角色不能瞬移
- 镜头N+1的角色位置应在镜头N位置的**合理移动范围**内
- 如果不合理：插入过渡镜头或重新设计

```
❌ 镜头1: Maya在(3,1) → 镜头2: Maya在(8,5)
   没有过渡，观众困惑："她怎么过去的？"

✅ 镜头1: Maya在(3,1)
   镜头2: Maya走向(5,2)（跟拍，展示移动过程）
   镜头3: Maya到达(8,5)（广角，展示新位置）
```

## 五、动作连续性

### 5.1 动作衔接规则

```
上一个镜头的结束状态 = 下一个镜头的起始状态

镜头1结束：Maya举起手电到胸前（右臂弯曲90°）
镜头2起始：Maya的手电正在胸前（右臂弯曲90°）← 必须匹配！
```

### 5.2 动作拆分原则

复杂动作需拆分为多个镜头，每个镜头承担一个动作阶段：

```
动作：Maya推开门→进入→环视→发现照片→走近→触摸

镜头1: Maya推开门（中景，从门外拍）→ 门缓缓打开
镜头2: Maya踏入房间（全景，从室内拍门的方向）→ 脚跨过门槛
镜头3: Maya环视房间（POV，手持摇镜）→ 手电光扫过墙壁
镜头4: 手电光照到照片墙（特写，照片被照亮）
镜头5: Maya走近照片墙（中景，侧面跟拍）→ 缓慢走近
镜头6: Maya伸手触摸照片（特写，手指触碰被红笔划掉的脸）
```

### 5.3 匹配剪辑（Match Cut）

利用动作的相似性做跨镜头的流畅衔接：

```
镜头5结束：Maya伸手触摸照片（手指向前）
镜头6起始：闪回中Maya伸手触摸另一个物体（手指向前）
（利用手的动作相似性做时空跳跃）
```

## 六、道具位置一致性

### 6.1 道具锁定规则

关键道具在同一个场景中位置必须锁定：

```
场景：Maya的办公室

桌面道具布局（镜头1建立）：
┌─────────────────────┐
│ 咖啡杯(左上)  台灯(右上) │
│                          │
│ 文件夹(中)   电脑(右下)   │
│ 照片(左下)               │
└─────────────────────┘

后续所有镜头中的桌面道具位置必须与此一致
```

### 6.2 道具移动追踪

如果道具确实需要移动，必须有明确的移动指示：

```
镜头3：Maya拿起咖啡杯（从左上角拿起）
镜头4：咖啡杯现在在Maya手中（位置变化合理）
镜头5：咖啡杯放在电脑旁（新位置：右下角电脑旁）
```

### 6.3 道具连续性检查

| 检查项 | 方法 |
|--------|------|
| 同一场景中道具位置 | 对比第一个建立镜头和后续镜头 |
| 道具存在/消失 | 道具不应无原因出现/消失 |
| 道具状态（满/空/开/关） | 跨镜头的状态变化需合理 |
| 消耗品（烟/饮料/纸） | 数量/状态需连续变化 |

## 七、光影连贯性

### 7.1 光源方向锁定

同一场景中主光源方向必须一致：

```
场景光源建立：月光从破碎的彩窗射入（光源在北方，从上向下）

镜头1（ELS）：月光从北→南射入，在地面形成彩窗投影
镜头2（MS）：Maya面部被北侧光源照亮（主光来自左侧/北）
镜头3（CU）：她的手被同样的北侧光照亮

❌ 镜头4（CU）：Maya面部被右侧光照亮（光源突然在南侧！）
```

### 7.2 光源变化标注

如果光源确实变化（如角色移动或光源本身变化），必须明确标注：

```
镜头5：Maya走过彩窗投影（光线在她身上扫过）
镜头6：她走到灯笼旁（光源切换：月光→灯笼暖光，光源方向改变）
标注：[光源变化：MOONLIGHT(N)→LANTERN(地面, 暖)]
```

### 7.3 光线质量一致性

同一场景中光质（Hard/Soft）和色温必须一致：

| 检查项 | 标注 |
|--------|------|
| 光质 | HARD / SOFT / MIXED |
| 色温 | 3200K(暖) / 5600K(日光) / MIXED |
| 光源 | MOON / NEON / TUNGSTEN / DAYLIGHT / FLUORESCENT |
| 特殊光效 | FLICKER / PULSE / GOD RAYS / VOLUMETRIC |

## 八、连贯性检查清单

### 每个场景结束时的检查：

```
📋 连贯性检查清单 — 场景[编号]

空间连贯性：
├── [ ] 所有镜头的相机位置是否在轴线的同一侧？
├── [ ] 如有越轴，是否有中性过渡镜头？
├── [ ] 角色位置跨镜头追踪是否合理？
└── [ ] 关键道具位置是否一致？

视线连贯性：
├── [ ] 对话镜头中的视线方向是否匹配？
├── [ ] 高低差视线是否正确？
└── [ ] POV镜头的视线方向是否与角色朝向一致？

动作连贯性：
├── [ ] 相邻镜头的动作是否衔接（结束状态=开始状态）？
├── [ ] 复杂动作是否合理拆分？
└── [ ] 动作方向和速度是否一致？

光影连贯性：
├── [ ] 主光源方向跨镜头是否一致？
├── [ ] 光质（硬/软）是否一致？
├── [ ] 色温是否一致（除非有灯光变化）？
└── [ ] 特殊光效是否连续？

时间连贯性：
├── [ ] 镜头间的时间流逝是否合理？
├── [ ] 同一场景是否为CONTINUOUS时间？
└── [ ] 如有时空跳跃，是否有明确标记？

情绪连贯性：
├── [ ] 相邻镜头的情绪差异是否≤2级？
├── [ ] 情绪转折是否有合理过渡？
└── [ ] 情绪节奏是否与叙事节奏匹配？
```

## 九、连贯性违规处理

### 违规等级

| 等级 | 符号 | 说明 | 处理 |
|------|------|------|------|
| 🔴 Critical | 🚫 | 明显的连贯性错误，观众会直接察觉 | 必须修正 |
| 🟡 Warning | ⚠️ | 潜在的连贯性问题，敏感的观众可能察觉 | 建议修正 |
| 🔵 Info | ℹ️ | 轻微的连贯性偏差，几乎不可察觉 | 标记供审查 |

### 常见违规及修正

| 违规 | 等级 | 修正方案 |
|------|------|---------|
| 直接越轴无过渡 | 🔴 | 插入中性过渡镜头（正面/背面） |
| 角色瞬移 | 🔴 | 插入移动过渡镜头 |
| 视线方向不匹配 | 🔴 | 调整角色朝向描述 |
| 道具消失/出现 | 🟡 | 添加道具出现/消失的镜头解释 |
| 光源方向微偏 | 🟡 | 标注光源方向，检查是否合理（角色移动导致） |
| 情绪跳跃>5级 | 🟡 | 插入情绪过渡镜头 |
| 动作起始状态略有不匹配 | 🔵 | 微调动作描述 |

## 十、分镜表中连贯性元数据格式

在分镜表中嵌入以下连贯性元数据：

```
📐 连续性卡片 — S[场景编号]-[镜号]

空间参数：
├── AXIS: [A↔B / SINGLE / NONE]
├── CAM_SIDE: [NORTH / SOUTH / EAST / WEST]（相对于轴线）
├── CHAR_A_POS: (x, y, z) 朝向:[N/S/E/W/NE/...]
├── CHAR_B_POS: (x, y, z) 朝向:[N/S/E/W/NE/...]
└── CROSS_AXIS: [NO / YES(有过渡) / ⚠️YES(无过渡)]

视线参数：
├── CHAR_A_EYELINE: [→/←/↑/↓/OFF-SCREEN/DIRECT]
├── CHAR_B_EYELINE: [→/←/↑/↓/OFF-SCREEN/DIRECT]
└── EYELINE_MATCH: [✅ / ❌]

动作参数：
├── ACTION_START: [延续自镜头N的结束状态]
├── ACTION_END: [传递给镜头N+1的起始状态]
└── ACTION_CONT: [✅ / ❌ / ⚠️]

光影参数：
├── KEY_LIGHT_DIR: [N/S/E/W/NE/...]
├── LIGHT_QUALITY: [HARD/SOFT/MIXED]
├── COLOR_TEMP: [3200K/5600K/MIXED]
└── LIGHT_CONT: [✅ / ⚠️变化(有原因)]

时间参数：
├── TIME_TYPE: [CONTINUOUS / JUMP(+Ns) / FLASHBACK / FLASHFORWARD]
└── TIME_CONT: [✅ / ⚠️]

连贯性总评：
├── 违规：[无 / 🔴N个 🟡M个 🔵K个]
└── 评级：[PASS / PASS WITH WARNINGS / NEEDS FIX]
```
